## рџЏ—пёЏ System Architecture

The system utilizes a directed acyclic graph (DAG) to pass a shared `ResearchState` between autonomous nodes. 

```mermaid
graph TD
    subgraph Orchestration [Multi-Agent Orchestration - LangGraph]
        A[User Query] --> B(Planner Agent)
        B -->|Generates JSON Search Strategy| C(Researcher Agent)
        C -->|Query| D[(ChromaDB Local Vector Memory)]
        D -->|Cached Data| C
        C -->|Scrape if Cache Miss| E[Tavily Web Search API]
        E -->|Live Data| C
        C -->|Raw Intelligence| F(Writer Agent)
        F -->|Synthesized Markdown| G[Final Executive Report]
    end
```

# рџ§  Persistent Memory Core

The system integrates **SentenceTransformers + ChromaDB** to provide long-term semantic memory for agents.

## Memory Workflow
1. Research findings are embedded into vector representations
2. Vectors are stored inside a persistent ChromaDB collection
3. Incoming queries are semantically matched against previous memories
4. Cached intelligence is reused when similarity thresholds are met

## Benefits
- Dramatically reduces redundant API calls
- Near-instant responses for repeated queries
- Lower operational costs
- Faster research cycles
- Context-aware intelligence retrieval

# рџ› пёЏ Engineering Challenges Solved

## вљ пёЏ Open-Source Tool Calling Instability
While working with open-source LLM orchestration, backend parser inconsistencies caused hallucinated legacy XML function tags such as:

```xml
<function>
```

### Solution
- Engineered strict negative constraints in system prompts
- Stabilized structured JSON routing
- Optimized prompting specifically for `Llama-3.1-8b-instant`

### Result
Reliable autonomous agent communication with deterministic outputs.

---

## вљЎ API Rate Limits & Context Window Constraints

Groq's ultra-fast LPU inference occasionally triggered `429 RateLimitErrors` during rapid autonomous execution cycles.

### Solution
- Implemented programmatic throttling using:
```python
time.sleep()
```

- Added strict Tavily response truncation logic
- Optimized prompt compression strategies
- Managed the ~6,000 token context window efficiently

### Result
Stable multi-agent execution without overflow or API failure.

---

## рџ§  Vector Database Caching Optimization

Repeated research queries unnecessarily increased latency and API consumption.

### Solution
Built a persistent `agent_memory_db` capable of:
- Semantic query routing
- Memory recall
- Cached intelligence retrieval
- Similarity-based context injection

### Result
Massive speed improvements and significantly reduced operational cost.
